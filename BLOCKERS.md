## BLOCKER-003: AI Engine Service Account Token

**Impact**: `ai-engine` cannot authenticate to collab-server.

After BLOCKER-001 env vars are set, generate the token:

```bash
node -e "
  const jwt = require('jsonwebtoken');
  const token = jwt.sign(
    { userId: 'ai-engine', email: 'ai@plugin11.internal', name: 'AI Engine' },
    process.env.JWT_SECRET,
    { expiresIn: '365d' }
  );
  console.log(token);
"
```

Set output as `COLLAB_TOKEN` in `ai-engine/.env`.

The collab-server `onAuthenticate` now bypasses workspace membership checks for `userId === 'ai-engine'` — this is the correct security gate.

## BLOCKER-004: WebContainer CORS Headers

**Impact**: `PreviewPanel` WebContainer iframe cannot call internal API routes.

Add to `apps/web/next.config.js`:

```js
async headers() {
  return [
    {
      source: '/((?!api/auth).*)',
      headers: [
        { key: 'Cross-Origin-Embedder-Policy', value: 'require-corp' },
        { key: 'Cross-Origin-Opener-Policy', value: 'same-origin' },
      ],
    },
  ];
},
```

**WARNING**: Test Google/GitHub OAuth popup flow after applying — COEP/COOP can break OAuth redirects.

## BLOCKER-007: Yjs Document GC / Compaction

**Impact**: `yjsDocument.data` blobs grow unboundedly over time.

Schedule a weekly cron to compact documents:

```ts
const compacted = Y.encodeStateAsUpdate(doc);
await prisma.yjsDocument.update({
  where: { name: documentName },
  data: { data: Buffer.from(compacted) },
});
```

Consider adding `updatedAt` index to `yjsDocument` and archiving records not touched in >90 days.
